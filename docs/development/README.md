# Development documentation

Roadplanner uses a specification-first, patch-based workflow designed to work from an iPad with GitHub Codespaces.

## Start here

1. Read the [AI Development Contract](../../AI_DEVELOPMENT_CONTRACT.md).
2. Read the relevant [Architecture Decision Records](../architecture/adr/README.md).
3. Create a task from [TASK_TEMPLATE.md](TASK_TEMPLATE.md).
4. Follow [DEVELOPMENT_WORKFLOW.md](DEVELOPMENT_WORKFLOW.md).
5. Apply deliveries using [PATCH_WORKFLOW.md](PATCH_WORKFLOW.md).
6. Validate against [DEFINITION_OF_DONE.md](DEFINITION_OF_DONE.md).

## Planning sources

- Product direction: [`ROADMAP.md`](../../ROADMAP.md)
- Prioritized work: [`BACKLOG.md`](../../BACKLOG.md)
- Concrete implementation contract: one `RP-XXX` task specification
- Reproducible public bugs: GitHub Issues when useful

Roadmap, backlog, task specifications, and issues have different purposes and must not become competing sources of truth.
